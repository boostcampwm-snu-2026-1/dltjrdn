# todo — 남은 작업 설명

프로젝트(마인크래프트 스킨 생성기)를 완성하기까지 **무엇이 남았고, 왜 필요하고, 어떻게 하는지**를 풀어서 설명한다.
checklist가 "현황 표시"라면 todo는 "남은 일 가이드"다.

## 할 일

1. `PLAN.md`와 `CLAUDE.md`를 읽어 목표 파이프라인(① 프롬프트 → ② SDXL → ③ 검증 → ④ 웹UI)을 파악한다.
2. 실제 파일 상태를 Glob/Read로 확인해 **아직 없거나 미완성인 부분**을 찾는다.
3. 남은 작업을 우선순위 순으로 정리하되, 각 항목마다:
   - **무엇을** 만들어야 하는지
   - **왜** 필요한지 (파이프라인에서의 역할)
   - **어떻게** 시작하는지 (구체적 첫 단계, 관련 파일/함수)
4. 사용자가 "다음 하나만 하자"고 하면 가장 위 항목부터 착수를 제안한다.

## 출력 형식

```
## 🎯 목표까지 남은 일

### 1. (가장 시급한 작업)
- 무엇: ...
- 왜: ...
- 어떻게: ... (첫 단계)

### 2. (다음 작업)
...

## 한 줄 추천
"지금은 N번부터 하면 돼" + 바로 시작할지 묻기
```

## 규칙
- 막연한 "UI 만들기" 식 말고 **구체적 첫 행동**까지 제시한다 (어떤 파일에 어떤 함수부터).
- 졸업작품 기여 포인트(① 프롬프트 엔지니어링, 품질 평가)를 약하게 두지 않도록 상기시킨다.
- 이미 끝난 일은 다시 적지 않는다. 실제 파일을 확인해 사실 기반으로 말한다.
